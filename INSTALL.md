## 1. 요구 사항

| 항목 | 버전 |
|------|------|
| Minecraft Java Edition | 1.21.4 |
| Fabric Loader | 0.16.10 이상 |
| Fabric API | 0.118.0+1.21.4 이상 |
| Java | 21 이상 |

> **주의:** Forge, NeoForge 환경에서는 동작하지 않습니다. 반드시 **Fabric** 환경이어야 합니다.

## 2. 설치 방법

### 2-1. Fabric 설치

Fabric이 설치되어 있지 않다면 먼저 설치해야 합니다.

1. [https://fabricmc.net/use/installer/](https://fabricmc.net/use/installer/) 접속
2. Minecraft 버전 `1.21.4` 선택 후 설치
3. 런처에서 Fabric 1.21.4 프로필로 실행 확인

### 2-2. Fabric API 설치

1. [https://modrinth.com/mod/fabric-api/versions?g=1.21.4](https://modrinth.com/mod/fabric-api/versions?g=1.21.4) 접속
2. `0.118.0+1.21.4` 이상 버전 다운로드
3. 다운로드한 JAR 파일을 `mods` 폴더에 복사

### 2-3. 창고지기 모드 설치

1. `changojigi-fabric-mod-1.0.0.jar` 파일을 `mods` 폴더에 복사

### 2-4. mods 폴더 위치

| 운영체제 | 경로 |
|---------|------|
| Windows | `%APPDATA%\.minecraft\mods\` |
| macOS | `~/Library/Application Support/minecraft/mods/` |
| Linux | `~/.minecraft/mods/` |

### 2-5. 최종 mods 폴더 구성 확인

```
mods/
├── fabric-api-0.118.0+1.21.4.jar      ← Fabric API
└── changojigi-fabric-mod-1.0.0.jar    ← 창고지기 모드
```

## 3. 최초 설정

### 3-1. 게임 실행

모드가 정상 설치되면 게임 실행 시 자동으로 설정 파일이 생성됩니다.

```
config/chestbot.json
```

### 3-2. 섬 참여 코드로 연결

Discord 봇에서 발급받은 **6자리 참여 코드**로 섬에 연결합니다.

```
/창고봇 연결 <참여코드>
```

예시:
```
/창고봇 연결 GCY3MG
```

연결에 성공하면 채팅창에 연결된 섬 이름이 표시됩니다.
이후 게임을 재시작해도 **자동으로 연결**됩니다.

## 4. 상자 등록 (관리자)

상자를 처음 등록하거나 수정하려면 관리자 권한이 필요합니다.
관리자 코드는 Discord 봇에서 발급받습니다.

### 4-1. 관리자 모드 진입

```
/창고봇 관리자 <관리자코드>
```

### 4-2. 상자 등록

1. 명령어로 등록 대기 시작:
   ```
   /창고봇 추가 <상자이름>
   ```
   예시: `/창고봇 추가 공용상자`

2. 등록할 **상자를 클릭**하면 자동으로 위치가 저장됩니다.

3. 여러 상자를 연속으로 등록하려면 위 과정을 반복합니다.

4. 모든 등록이 완료되면 서버에 자동으로 저장됩니다.

### 4-3. 등록 취소

등록 대기 중인 상자를 취소하려면:

```
/창고봇 제거 <상자이름>
```

## 5. 명령어 목록

모든 명령어는 한글(`/창고봇`)과 영문(`/chestbot`) 모두 사용 가능합니다.

### 일반 명령어

| 한글 명령어 | 영문 명령어 | 설명 |
|------------|------------|------|
| `/창고봇 연결 <코드>` | `/chestbot connect <code>` | 참여 코드로 섬 연결 |
| `/창고봇 목록` | `/chestbot list` | 등록된 상자 목록 표시 |
| `/창고봇 새로고침` | `/chestbot reload` | 서버에서 설정 재로드 |
| `/창고봇 입금사유 <사유>` | `/chestbot depositreason <reason>` | 은행 거래 사유 입력 |

### 관리자 명령어

| 한글 명령어 | 영문 명령어 | 설명 |
|------------|------------|------|
| `/창고봇 관리자 <코드>` | `/chestbot admin <code>` | 관리자 모드 진입 |
| `/창고봇 추가 <이름>` | `/chestbot add <name>` | 상자 등록 대기 시작 |
| `/창고봇 제거 <이름>` | `/chestbot remove <name>` | 등록 대기 중인 상자 취소 |

## 6. 은행 거래 기록

섬 은행에서 입금 또는 출금이 발생하면 자동으로 감지합니다.

### 동작 흐름

1. 섬 은행에서 입금/출금 발생
2. 채팅에 거래 메시지 출력
3. 창고지기가 메시지를 감지하고 사유 입력 안내
4. 아래 명령어로 사유 입력:
   ```
   /창고봇 입금사유 <사유>
   ```
5. Discord 봇에 거래 내역이 자동 기록됨

> 사유를 입력하지 않아도 거래 내역은 기록되지만, 사유란이 비어 있게 됩니다.

## 7. 설정 파일

설정 파일 위치: `config/chestbot.json`

```json
{
  "server_url": "http://서버주소:5000",
  "island_code": "참여코드"
}
```

| 항목 | 설명 | 기본값 |
|------|------|--------|
| `server_url` | 창고지기 백엔드 서버 주소 | 서버 운영자에게 문의 |
| `island_code` | 연결된 섬의 참여 코드 | `/창고봇 연결` 명령어로 자동 저장 |

> 일반적으로 직접 편집할 필요 없이 인게임 명령어로 자동 관리됩니다.

## 8. 자주 묻는 질문

**Q. 게임 실행 시 "호환되지 않는 모드가 있습니다" 오류가 나와요.**
A. Fabric API가 설치되지 않은 경우입니다. [2-2 Fabric API 설치](#2-2-fabric-api-설치) 항목을 확인하세요.

**Q. `/창고봇 연결` 명령어가 작동하지 않아요.**
A. 서버에 접속한 상태에서만 사용 가능합니다. 싱글플레이 또는 멀티플레이 서버에 입장한 후 시도해주세요.

**Q. 상자를 열어도 Discord에 기록이 안 돼요.**
A. 다음을 순서대로 확인하세요:
1. `/창고봇 목록` 명령어로 해당 상자가 등록되어 있는지 확인
2. 섬이 연결되어 있는지 확인 (서버 재접속 후 자동 연결 여부)
3. Discord 봇 서버가 정상 실행 중인지 확인

**Q. 참여 코드는 어디서 받나요?**
A. Discord 서버에서 봇에게 `/창고 설정` 명령어를 사용하면 섬이 생성되고 참여 코드가 발급됩니다. 서버 관리자에게 문의하세요.

**Q. 모드가 싱글플레이에서도 동작하나요?**
A. 동작은 하지만, 백엔드 서버와 통신이 되어야 기록이 저장됩니다. 백엔드 서버가 없다면 기록이 전송되지 않습니다.
