# chest-fabric-mod SETUP

이 문서는 **단일 섬용 `chest-discord-bot` 인스턴스**에 연결하는 기준으로 작성되었습니다.

- 일반 연결은 **서버 주소 기반**입니다.
- 창고 등록은 여전히 **Discord에서 발급한 관리자 코드**가 필요합니다.
- 즉, **참여 코드는 더 이상 없지만 관리자 코드는 있습니다.**

## 0. 누가 무엇을 하나요?

### 운영자

- 봇 실행
- `/창고 설정`, `/창고 채널연결` 수행
- 필요 시 `/창고 관리자코드` 발급
- 플레이어에게 서버 주소 전달

### 플레이어

- `/창고봇 서버 <서버주소>` 입력
- `/창고봇 연결` 실행

### 인게임 관리자

- `/창고봇 관리자 <관리자코드>`
- `/창고봇 추가 <창고이름>` 후 상자 클릭

## 1. 준비물

- Minecraft `1.21.4`
- Fabric Loader
- Java 21
- 실행 중인 `chest-discord-bot`
- 봇 운영자가 알려준 서버 주소

## 2. 설치

1. Fabric 환경을 준비합니다.
2. 빌드한 모드 jar를 `mods/` 폴더에 넣습니다.
3. 게임을 한 번 실행합니다.

## 3. 서버 주소 연결

게임에 들어가서 아래 순서대로 입력하세요.

```text
/창고봇 서버 <서버주소>
/창고봇 연결
```

예시:

```text
/창고봇 서버 https://your-bot-domain.example.com
```

주소는 반드시 `http://` 또는 `https://`로 시작해야 하며, **모드가 실행되는 환경에서 실제 접근 가능해야 합니다.**

성공하면 현재 섬 이름과 chest 개수가 출력됩니다.

## 3-1. 첫 실행 순서

1. 운영자에게 서버 주소를 받습니다.
2. `/창고봇 서버 <서버주소>` 입력
3. `/창고봇 연결` 실행
4. `/창고봇 목록`으로 창고 목록 확인
5. 창고 등록이 필요하면 Discord에서 `/창고 관리자코드` 발급
6. 게임에서 `/창고봇 관리자 <코드>` 실행

## 4. 주요 명령어

### 일반 사용자

- `/창고봇 서버 <주소>`: 백엔드 서버 주소 저장
- `/창고봇 연결`: 현재 서버 설정 동기화
- `/창고봇 목록`: 등록된 창고 목록 확인
- `/창고봇 새로고침`: 서버에서 설정 다시 불러오기
- `/창고봇 입금사유 <사유>`: 은행 거래 사유 전송

### 관리자

- `/창고봇 관리자 <코드>`: 관리자 모드 진입
- `/창고봇 추가 <창고이름>`: 등록할 창고 이름 지정
- 실제 상자를 우클릭: 해당 이름으로 저장
- `/창고봇 제거 <창고이름>`: 등록 대기 목록에서 제거

## 5. 설정 파일

경로:

```text
<게임 디렉터리>/config/chestbot.json
```

예시:

```json
{
  "server_url": "https://your-bot-domain.example.com"
}
```

## 6. 정상 동작 체크

- `/창고봇 서버` 입력 후 주소가 저장되어야 합니다.
- `/창고봇 연결` 시 섬 이름과 chest 수가 보여야 합니다.
- `/창고봇 목록`에 서버에 저장된 창고가 보여야 합니다.
- 상자 사용 후 Discord 로그 채널에 로그가 와야 합니다.

## 7. 문제 해결

### 연결 실패

- 서버 주소가 `http://` 또는 `https://`로 시작하는지 확인하세요.
- 봇 서버가 실행 중인지 확인하세요.
- 서버 주소와 포트가 외부에서 접근 가능한지 확인하세요.

### 목록이 비어 있을 때

- 서버 쪽에 아직 창고 설정이 없는 상태일 수 있습니다.
- 운영자에게 `/창고 현재설정`과 채널 연결 상태를 확인해 달라고 하세요.

### 관리자 모드가 안 될 때

- 먼저 `/창고봇 연결`로 현재 설정을 불러왔는지 확인하세요.
- Discord에서 **새 관리자 코드**를 다시 발급받아 사용하세요. 관리자 코드는 1회성/유효기간 제한이 있을 수 있습니다.
- 봇 서버가 정상 응답 중인지 확인하세요.

```

```
